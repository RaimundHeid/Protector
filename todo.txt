# Performance Optimizations for NNUE (Inspired by Stockfish)

Listed in order of likely performance impact.

1. **Double Incremental Updates (Medium-High Impact)**
   - Apply the add/sub feature lists for two consecutive moves in a single pass over the
     accumulator, deferring the intermediate `memcpy` until the result is actually needed.
   - This is distinct from computing two moves "at once"; the gain comes from eliminating one
     full accumulator copy on the critical path.

2. **Advanced SIMD & VNNI (Medium-High Impact)**
   - Add AVX-VNNI (`_mm256_dpbusd_epi32`) and AVX-512-VNNI support; detect them separately
     at runtime since they have different CPUID bits.
   - Use NEON dot-product (`vdotq_s32`) on ARM for the L2/L3 fully connected layers.
   - Replace C++-style template specialization references with macro-based or `_Generic`
     dispatch (this is a C project).

3. **Weight Permutation & Implicit Clipping (Medium Impact)**
   - Pre-permute Feature Transformer weights to match the SIMD layout expected by `packus`.
   - `_mm_packus_epi16` is already used at the narrow step in `transform_small`/`transform_big`,
     but clipping is still done with explicit `_mm256_min_epi16` / `_mm256_max_epi16` pairs.
     Restructure so that `packus` performs the clip implicitly, removing those two instructions
     per loop iteration.
   - The activation is SCReLU (Squared Clipped ReLU): clip to [0, 255], multiply the two
     halves, shift right by 9. Use this consistent name throughout comments and documentation.

4. **Batched fc_u8s8 / Tiled Output (Medium Impact)**
   - `fc_u8s8` currently computes each output neuron independently, re-loading the full input
     vector for every output.
   - Process 4–8 output neurons simultaneously so the input vector is reused while it is
     still hot in SIMD registers, improving utilisation for the big net's L2 (31 outputs)
     and L3 (32 outputs).

5. **SIMD PSQT Updates (Low-Medium Impact)**
   - Integrate PSQT accumulation into the main Feature Transformer SIMD loops instead of
     using separate scalar loops, reducing loop-control overhead during incremental updates.

6. **Prefetching in Search (Low-Medium Impact)**
   - Insert `__builtin_prefetch` / `_mm_prefetch` hints before loading the next ply's
     Accumulator during tree traversal to hide memory latency for the large (~8.5 KB)
     accumulator structs.

7. **Tiling & Register Optimization (Low Impact)**
    - Use architecture-specific tiling in the L2/L3 loops to ensure working data fits in
      SIMD registers, minimising cache-line spills during accumulation.
